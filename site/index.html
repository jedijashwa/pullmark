Docs are meant to be diffed read.

PullMark renders your local Markdown the way readers will see it — and shows documentation-heavy GitHub pull requests as rendered diffs, with just the changed words highlighted. Comment, suggest, and submit your review without ever squinting at a wall of +/− prose.

why

Prose isn't code.
Stop reviewing it like code.

More and more of what flows through pull requests isn't code — it's documents: design docs, decision records, agent and skill definitions, runbooks, READMEs. Reviewing those as source lines is the wrong altitude. A document is meant to be read the way readers will see it — tables as tables, diagrams as diagrams. A source diff of prose actively hides what changed.

PullMark splits old and new into blocks, aligns them, and renders the result with changed blocks marked in place. When an edit is small, you get a word-level diff inside formatted text — a one-word edit shows as one highlighted word. And because every block keeps its original line numbers, review comments land exactly where GitHub expects them.

“The reading room next to the workshop.”
Note

PullMark is deliberately not a full review client. It shows only the Markdown files in a PR — for the code parts of a change, your usual review tools remain the right place.

reviewing

The whole review, from the rendered page.

Switch between inline and side-by-side layouts, drop to the raw Source Diff, or preview the final Result — all from the toolbar. YAML front matter renders as a quiet metadata table, not a wall of bold prose — so config-heavy docs diff cleanly too. Existing review threads appear under the blocks they discuss. Hover any block, click the bubble, and reply — or start something new. Open PRs are checked every minute; if the branch moves, one click refreshes and your drafts survive.

Comment Now

Posts a single review comment immediately, anchored to the exact lines behind the rendered block.

Add to Review

Collects drafts locally. Submit them together — Comment, Approve, or Request Changes — or save a pending review to finish on github.com.

Insert Suggestion

Pre-fills a ```suggestion block with the targeted lines, so the author applies your edit in one click.

reading

A proper reader for the Markdown already on your disk.

Open files or whole folders; everything re-renders the moment it changes on disk. Relative images resolve, links to other Markdown files open in-app, heading anchors jump within the page — and everything GitHub renders, PullMark renders, verified construct-by-construct in CI.

  • History in the margin — a quiet blame gutter: one avatar per run of blocks, hover for the author, date, and commit, click for the line history — for local files and PR docs alike.
  • Front matter as metadata — YAML front matter renders as a collapsed key/value table at the top of the page, not a wall of bold prose.
  • Live re-render — save from any editor, the page updates.
  • Math & [toc] — $…$ and $$…$$ render through bundled KaTeX, fully offline; a lone [toc] becomes a linked table of contents.
  • Outline sidebar — a navigator-style heading outline, one toggle away.
  • Find in page — ⌘F with match stepping; ⇧⌘F searches every file in the sidebar, results grouped with context.
  • Export & copy — Export as PDF or a self-contained HTML file; ⌘C copies rich text that pastes formatted, ⌥⌘C copies the original Markdown source.
  • Word count — a quiet words · minutes pill on every rendered document.
  • Link previews — a status pill shows every destination on hover.
  • Light & dark — follows the system, restyles everything, Mermaid included.
  • Reading themes — Editorial (serif headers on warm paper), GitHub (the classic github.com look), or Terminal (monospace, phosphor accent), with live previews in Settings — or drop .css files into the Themes folder to add your own.
  • Update Now — when a release ships, brew-managed installs upgrade and relaunch from a single banner button, and a What's New sheet catches you up after.

at-home-on-your-mac

Native, in all the ways that count.

Quick Look

Press space on any Markdown file in Finder and see it rendered PullMark-style, before you ever open an app.

A real CLI

pullmark README.md, pullmark ~/notes docs/ — files and folders open in the running app.

Default app for .md

One click in Settings makes double-click in Finder mean rendered, everywhere — and if an upgrade ever drops the binding, PullMark notices and offers it back.

Your existing credentials

No separate login. PullMark uses gh auth token or your git credential helper — private and org repos just work.

Nothing left on disk

Fetched PR content lives in ephemeral sessions and bounded memory caches — nothing from GitHub is written to persistent storage.

Open source, MIT

Swift, zero package dependencies, and a license that stays out of your way. Read the source.
